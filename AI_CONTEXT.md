# AI Context - GenieGoods Frontend

## 🤖 AI 활용 개발 원칙

본 프론트엔드 프로젝트는 Cursor를 활용하여 AI와 협업하는 방식으로 개발되었습니다.

- 반복적인 UI 컴포넌트 및 구조는 AI를 통해 생성
- 개발자는 사용자 흐름과 상태 관리 설계에 집중
- 생성된 코드는 반드시 검증 후 수정하여 반영

👉 AI를 단순 도구가 아닌, 작업을 위임하고 결과를 검증하는 협업 대상으로 활용합니다.

---

## 📌 프로젝트 개요

GenieGoods 서비스의 프론트엔드는 사용자가 반려동물 이미지를 업로드하고, AI가 생성한 굿즈 시안을 확인 및 주문할 수 있는 인터페이스를 제공합니다.

---

## 🧩 주요 역할

- 이미지 업로드 및 결과 확인 UI
- 사용자 인증 및 상태 관리
- API 연동 및 데이터 처리
- 사용자 경험(UI/UX) 개선

---

## ⚙️ 기술 스택

- React (Vite)
- TailwindCSS
- Zustand (전역 상태)
- TanStack Query (서버 상태 관리)
- Axios (API 통신)
- React Router

---

## ⚠️ AI 코드 검증 원칙

- 비동기 요청 흐름 (loading / error / success 상태) 검증 필수
- API 응답 실패 및 예외 처리 UI 구현
- 사용자 입력값 검증 (빈 값, 잘못된 형식 등)
- 상태 관리 로직의 일관성 유지

---

## 📝 작업 방식

- TODO.md → 구현해야 할 기능 정의
- REVIEW.md → 코드 검증 및 개선 기록

---

## 💡 핵심 개발 방식 요약

AI에게 UI 및 로직 생성을 맡기고,  
저는 사용자 경험과 데이터 흐름 검증에 집중하여 개발했습니다.
